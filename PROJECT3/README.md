## 프로젝트 소개

본 프로젝트는 Flutter를 활용한 **할 일 관리(Todo) 애플리케이션**임
단순한 기능 구현을 넘어, 실제 사용 과정에서 발생하는 UI·UX 문제를 직접 분석하고
코드 수정과 구조 개선을 통해 해결하는 것을 목표로 진행했음

특히 입력 흐름, 상태 관리, 화면 간 데이터 동기화 등
실제 앱 사용성에 영향을 주는 부분에 집중하여 구현했음

## Trouble Shooting

본 프로젝트에서는 단순 구현을 넘어,
실제 개발 과정에서 마주한 문제들을 직접 해결하는 데 중점을 두었음

### 1. 코드 가독성 및 유지보수성 개선

각 위젯과 로직의 역할이 명확히 드러나도록 주석을 추가하고,
불필요하게 복잡했던 위젯 구조를 정리했음

이를 통해 코드 가독성과 이후 유지보수성을 함께 개선했음

### 2. 에셋 이미지 미표시 문제 해결

에셋 이미지를 등록했음에도 화면에 이미지가 표시되지 않는 문제가 발생했음
문제를 분석한 결과, 단일 원인이 아닌 여러 설정 이슈가 복합적으로 겹친 문제였음을 확인했음

초기에는 pubspec.yaml 파일 내에 flutter: 키가 중복 선언되어 있어
에셋 설정이 정상적으로 로드되지 않는 상태였음.
이로 인해 이미지 경로가 올바르게 작성되어 있음에도 에셋을 인식하지 못했음

이후 수정 과정에서 pubspec.yaml 변경 사항이 저장되지 않은 채 실행되는 경우가 있었고,
실제 파일 위치와 코드에서 참조하는 에셋 경로가 일치하지 않는 시점들도 확인했음
Flutter가 에셋 경로를 엄격하게 비교하는 특성상,
경로가 일치하지 않을 경우 이미지가 로드되지 않음을 확인했음

이를 해결하기 위해

- pubspec.yaml 구조를 정리하여 flutter: 키를 단일 선언으로 유지하고
- 에셋을 폴더 단위로 등록하여 경로 관리 복잡도를 줄였으며
- 실제 파일 위치와 코드 내 경로를 일관되게 맞추고
- flutter clean 및 flutter pub get을 통해 캐시를 초기화했음


### 3. 입력창 UI 커스텀

기본 `TextField`의 하단 밑줄(Underline)이
과제 가이드의 디자인과 맞지 않는 문제를 확인했음

`InputDecoration`의
`border: InputBorder.none` 속성을 적용하여 기본 선을 제거함으로써
가이드 이미지와 동일한 깔끔한 입력창 UI를 구현했음

---

### 4. 레이아웃 버그 수정 (패딩 중복 문제)

초기 구현 시 `Padding`이 중복 적용되어
입력창 너비가 의도보다 지나치게 좁아지는 현상이 발생했음

레이아웃 구조를 점검한 결과 불필요한 외부 패딩이 원인임을 확인했으며,
중복 패딩을 제거하여 가이드와 동일한 너비를 확보했음

---

### 5. 키보드 포커스 유지 문제 해결

텍스트 입력 시 한 글자 입력할 때마다
키보드가 사라지는 UX 문제를 발견했음

위젯 리빌드 과정에서 입력 상태가 유지되지 않는 것이 원인이라 판단하여
`TextEditingController`를 도입함으로써
입력 상태를 안정적으로 관리하도록 수정했음

이를 통해 입력 중 키보드가 유지되는 자연스러운 입력 경험을 구현했음

---

### 6. 상세 페이지 즐겨찾기 처리 중 화면 전환 문제 해결

상세 페이지에서 즐겨찾기(별표)를 수정할 경우,
의도와 다르게 해당 화면이 즉시 종료되며 이전 화면으로 돌아가는 문제가 발생했음

원인을 분석한 결과,
즐겨찾기 상태 변경 시 Navigator.pop이 함께 호출되며
상세 페이지가 즉시 닫히는 구조로 구현되어 있었음을 확인했음

이를 해결하기 위해

즐겨찾기 토글 시에는 화면을 유지하도록 로직을 분리하고

상세 페이지 종료 시에만 Navigator.pop을 호출하도록 수정했음

그 결과,
상세 페이지에서 즐겨찾기 상태를 자유롭게 수정할 수 있으며
사용자가 의도했을 때만 이전 화면으로 이동하도록 UX를 개선했음
